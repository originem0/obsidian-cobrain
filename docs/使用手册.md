# Cobrain 使用手册

> 一句话：Cobrain 是面向创作者的 Obsidian AI 副脑。它会在你提问时自动找出 vault 里的相关旧笔记，让可定制人设的 AI 基于你的材料追问、推敲、组织思路，帮助你从旧经验里长出新的表达。

## 一、快速上手

1. **配三套 API**（设置 → 第三方插件 → Cobrain，折叠分区）：文本 LLM、图像、嵌入，各填 Base URL + Key，点「检测」从端点实际可用的模型里下拉选（文本可再点「测试」验证连通，嵌入按真实试嵌入筛选）。
2. **建第二大脑**：跑命令 `Cobrain: 重建索引`，把整个 vault 嵌入进去（走嵌入代理，vault 大会慢、可能 429，耐心/重试）。
3. **开面板**：左侧 ribbon 的脑图标，或命令 `Cobrain: 打开创作副脑`。
4. **继续或新开**：如果这个面板槽位有未结束草稿，会让你选「恢复草稿」或「新建空对话」。关闭面板不是删除对话；要彻底结束，用面板里的「清空」。

## 二、它怎么帮助创作

这里的创作不是让 AI 替你写文章，而是把你的经验、旧知识、当前问题、分析和评价综合起来，形成一个带有你自己判断的新表达。Cobrain 帮的是这条链路。

第一，它帮你调取旧材料。不是手动复制材料，而是每次提问都会从 vault 找出相关旧笔记。

第二，它制造旧知识和当前问题的碰撞。差异、失洽、意外、回溯常常会逼出新的洞见。Cobrain 的价值正是把你忘掉的旧笔记撞回眼前，让「我现在的问题」和「我过去写过的东西」发生冲突。

第三，它不是直接给答案，而是追问和推敲。创作需要问题意识，也需要评价能力。Cobrain 的人设 Prompt 可定制，你可以让它像助产士、批评者、编辑、外星人类学家一样工作。它可以追问你的前提，也可以用「推敲」检查读者吸引力、论证水平、洞见水平，帮你从作者切到评判者。

第四，它帮你组织思路。概念图不是装饰，它对应创作里的「概念地图」：把焦点问题、概念、关系摆出来，暴露缺口和断层。它不是替你思考，而是让思考变得可见。

第五，它把一轮思考保存回 vault。这不是最大卖点，但它是闭环。新的笔记会变成下一轮创作的旧材料，让你的创作系统越用越厚。

## 三、怎么用好它（最佳实践 —— 重点）

> 想直接看一遍**真实实录**（用「世界何以至此」库跑出来的完整对话 / 概念图 / 笔记），看链路怎么转起来 → [实践演练](实践演练.md)。下面是抽象原则版。

1. **当它是助产士，别当答案机。** 它每轮会回抛一个问题——**别跳过，先自己答一句再往下**。被问住的地方，就是你认知里的缝隙；卡住不是坏事，是你该下探的入口。
2. **先喂前知识。** 你 vault 越厚、写过的越多，它接得越准（「想要 AI 启发你，你得先启发 AI」）。空 vault 用它 ≈ 浪费。
3. **盯住「相关旧笔记」那一块。** 面板会把检索到的旧笔记摊出来——点开它们，常会有「我居然写过这个」的撞击（第二大脑的「联想」）。**这是 Cobrain 最值钱的地方，别划过去。**
4. **问好问题。** 用**差异性发问**：「A 和 B 差在哪」「为什么这里不一样」，而不是「X 是什么」。卡住时试**辩证逆转**：把障碍当条件——「会不会正是这个困扰，反而成全了那件事？」
5. **进入「比照—落差—改进」循环。** 对话 → `存为笔记` → 读着觉得不对劲（落差）→ 再问、再改。理解和作品是磨出来的，不是一次生成的。
6. **概念图：让它起头，你来连。** 价值在你**自己建构**的过程（发现缺口），不在它一键吐出的那张图。把它当草稿，自己改。
7. **配图：辅助，别较真。** 它不在方法论骨架里，可有可无，别指望它出彩。
8. **存为笔记 = 闭环，但别囤积。** 沉淀成带双链的笔记，它进了 vault，下次就能被检索撞出来。保存时会冻结当时的对话、来源、概念图和配图；同一份内容重复点会提示已保存过，只有对话或产物变了才保存新版本。

## 四、一个日常工作流

散步时冒出一个困惑 → 打开面板抛给它 → 看它**撞出的旧笔记** + **回抛的问题** → 来回几轮，逼自己把它想透 → `存为笔记`（自动双链回相关旧笔记）→ 隔天再聊别的，RAG 又会把今天这篇撞出来。日积月累，vault 长成你的第二大脑。

## 五、命令 & 设置速查

**命令**（Ctrl/Cmd+P）：
- `Cobrain: 打开创作副脑` — 开对话面板
- `Cobrain: 重建索引` — 从零重嵌整个 vault（换嵌入模型后必跑）
- `Cobrain: 测试检索` — 输入一句话，看检索命中（分数+路径+片段），用来判断索引质量
- `Cobrain: 查看索引状态` — 看已索引笔记数、chunk 数、当前嵌入模型、是否正在索引、最近更新时间和失败列表

**设置**（折叠分区）：三套端点 + key + 模型，每套都能「检测」模型并下拉（文本可「测试」连通）；笔记目录（默认 `cobrain-note`）、附件目录、标签；三个可编辑提示词（**副脑人设** / 概念图 / 笔记综述）；概念图方向(TD/LR)与详细度；配图风格/尺寸/质量。

**提示词很重要**：副脑人设决定它是逼你想，还是变成顺从的答案机；笔记综述决定保存的是思考入口，还是聊天存档。默认提示词在源码 [src/settings.ts](../src/settings.ts) 里的 `DEFAULT_TUTOR_PROMPT`、`DEFAULT_CONCEPT_MAP_PROMPT`、`DEFAULT_NOTE_PROMPT`。已安装使用时，设置面板里的文本会覆盖代码默认值；改源码后需要重新构建/安装，且不会覆盖你已经保存过的设置。

## 六、注意事项

- **隐私**：索引时**笔记全文**会发往嵌入代理；对话时历史 + 检索片段会发往文本 LLM 代理（详见 `README.md` 的「隐私边界」）。Key 只存本地 `data.json`，不入库。
- **换嵌入模型** → 旧索引维度不兼容，会自动清空，需重跑 `Cobrain: 重建索引`。
- **重建索引**会重新嵌入整个 vault（4 路并发，走代理仍可能 429，单篇失败不影响整轮，慢就重试）。
- **长对话上下文**：面板会保留草稿，每轮把最近 20 条消息原文发给模型；更早的消息会在后台自动压缩成滚动摘要、随对话一起发送，面板会提示压缩进度。
- **移动端**：桌面 / 移动端通用，但**移动端只读检索**——不自动重嵌、不写索引分片，索引一律在桌面端跑「重建索引」后随 vault 同步过去（跨设备同步设置见 `README.md`「跨设备使用」）。
- **已知问题**：向量索引按每篇笔记分片存在插件目录的 `index/` 下（int8 量化，约 273 篇 ≈ 10 MB）；改一篇只重写它那个小分片、不再整份重写，块级增量 diff 留待后续。
