## 演进（2026-06-19）：从「学习导师」转向「创作副脑」（Cobrain）

读完激发本项目的全部材料（汤质《高手的黑箱》）后，定位做了一次纠偏，并据此重命名为 **Cobrain（创作副脑）**：

- **不是「学习导师」(讲解员)，是「助产士」**：那套方法论是创作方法论，不是学习方法论。导师 prompt 从「拆解 / 按你水平讲」改为「先问后讲」——基于你写过的笔记回抛差异性问题 / 辩证逆转（依据 3.5、5.6），逼你自己想。
- **旧笔记从「静默注入」改为「显式撞你」**：检索命中的旧笔记不再只喂给模型，而是显式列在面板上、可点开——把第二大脑的「联想」做成体验本身（依据 2.4「死去的记忆在第二大脑中攻击你」）。
- **铁律：逼你想，不替你想**：方法论自身警告别让工具成为目的、别造「流动性机器」磨平「阻塞感」（2.4 / 7.5）。因此走「做减法」，不堆功能（曾否决一版加「逼问 / 推敲」模块的方案）。配图 / 概念图保留但不再是主角。
- **更名**：id `learning-tutor` → `cobrain`；显示名 → 创作副脑 / Cobrain；图标 → `brain`。

> 以下为 2026-06-18 的原始设计记录，框架（架构 / 数据流 / 错误处理 / 隐私边界）大体仍准；其中「学习导师 / 讲解」的取向已被上述演进取代，以「创作副脑 / 助产士」为准。

## 背景与目标

用户以 Obsidian 为主知识库，手握 gpt-image-2 图像 API 与文本 LLM API（含 embeddings），希望做一个帮助「学习」的工具。经多轮澄清，目标收敛为：

> 一个**独立的 Obsidian 插件**，作为「懂你 vault 的学习导师」——以对话方式学习一个主题，导师检索你已有笔记（RAG 闭环）、按你已知水平讲解、拆解概念，并在你**显式指令**下把内容沉淀为结构化笔记；视觉层以**概念图（Mermaid）为主、gpt-image-2 插画为辅**，自动链接回相关旧笔记，形成第二大脑学习闭环。

**参考材料背书**：用户当前在学 汤质《高手的黑箱》第 7 章。该课程方法论直接塑造了本工具取向——概念地图（焦点问题→概念→关系）、输入/输出黑箱式 agent、提问力 + 鉴赏力。注：汤质在 7.1 亲口推荐「Obsidian + Smart Connections」作第二大脑方案，印证了 RAG/语义连接方向的正确；但 SC 作为成品无法满足「自由度高 + 配图 + 自定义」的诉求，故自建。

## 需求

1. **形态**：Obsidian 插件（TypeScript），独立运行，不依赖 SC 主体。
2. **主交互**：侧边栏对话式学习（导师）。
3. **学习域**：混合（抽象概念 / 词汇 / 事实），按内容类型调整讲法与视觉策略；当前主力材料为密集哲学/概念内容。
4. **知识库（RAG 闭环）**：
   - 输入：检索 vault 已有笔记作上下文，按用户已知水平讲、避免重复、连接新旧。
   - 输出：结构化笔记写回 vault，双链到相关旧笔记。
   - 实现：**自建向量库**——**云端嵌入 API（OpenAI 兼容 `/embeddings`，可插拔 Embedder 接口，默认 `BAAI/bge-m3`）**嵌入笔记，向量存插件数据，不耦合 SC 内部格式。（实现期变更：最初设计为本地 transformers.js，后改用云端 embeddings API——更快、中文检索质量更好；**代价是笔记全文会发往嵌入代理，见下方「隐私边界」**。）
5. **生成时机**：**显式触发**（用户下命令才生成笔记/图），控制成本与质量。
6. **视觉层（分工）**：
   - **Mermaid 概念图（主）**：焦点问题→概念→关系，与用户共建，服务「自己画概念图」这一学习目标。
   - **gpt-image-2 插画（辅）**：仅对可具象的概念，生成视觉隐喻。
7. **可用 API（已确认实测）**：文本 LLM = `z-ai/glm-5.1`（`https://wududu.edu.kg/v1`，OpenAI 兼容，chat 实测 200）；图像 = `gpt-image-2`（`https://freeapi.dgbmc.top/v1`，OpenAI 兼容 HTTP）；**embeddings = 云端 OpenAI 兼容 API（默认 `BAAI/bge-m3`，端点/模型可配）**。密钥仅存插件设置，绝不入库。
8. **自由度**：高度可定制——提示词、视觉策略、笔记模板、（未来）多 agent 均可改。

## v1 范围（YAGNI）

**做**：
- 导师对话面板 + RAG 接地
- 导师提示词：拆解 / 按水平讲 / 引用检索到的笔记 / 苏格拉底追问
- 三个显式命令：`存为笔记`、`概念图`（Mermaid）、`配图`（gpt-image-2）
- 设置页：各 API 端点/key/模型、笔记目录、附件目录

**不做（v2+）**：课程式多 agent 工作流（漫游者/探险家/扩写者…）、间隔重复复习、多模型对比、TTS。

> 先把「对话 → 拆解 → 存成带图带链的笔记」这条主线跑通。

## 架构与组件

纯 TS Obsidian 插件。SC 可与之并存（用户若想用其侧栏），但本插件不依赖它。

| 组件 | 职责 |
|---|---|
| Chat 面板（ItemView 侧栏） | 导师对话 UI |
| LLM 客户端 | 调用户文本 API（OpenAI 兼容，baseURL+key+model 可配） |
| 检索器（RAG） | 自建向量库：增量嵌入 vault 笔记，查询时取 top-K |
| 导师控制器 | 系统提示词，组织拆解 / 接地 / 追问 |
| 笔记写入器 | 生成结构化 md（frontmatter + 正文 + 概念图 + 配图 + `[[双链]]`）写入指定目录 |
| Mermaid 生成器 | LLM 出 Mermaid，先校验后渲染 |
| 配图生成器 | 隐喻提示词模板调 gpt-image-2 → 存附件 → 嵌入 |
| 向量索引存储 | 嵌入向量 + 笔记元数据持久化在插件 data |
| 设置页 | API / 目录 配置 |

## 数据流（一次学习会话）

1. 用户在面板抛出话题/疑问。
2. 检索器嵌入查询 → 取 vault top-K 相关笔记 → 作上下文。
3. 导师 LLM 接地回答（拆解、按你水平、点名相关旧笔记）。
4. 苏格拉底式往返。
5. 显式命令：`概念图`→Mermaid；`配图 X`→gpt-image-2；`存为笔记`→写文件 + 回链。
6. 笔记落进 vault、连进关系图；下次会话 RAG 即可见 → 闭环。

**索引维护**：首次全量嵌入；之后监听 vault 变更增量更新（增/改/删）。

## 关键设计决策

- **架构选 X（独立插件）** 而非 Y（寄生 SC）/ Z（fork SC）：因「自由度高」——SC 是可配置成品而非可编程平台，无法内嵌配图、自定义 agent、深改导师行为。
- **RAG 自建向量库** 而非复用 SC 索引：避免耦合 SC 内部格式（其升级易破），换取自由与稳健；用户 API 含 embeddings，成本可接受。
- **视觉分工**：Mermaid 扛抽象结构（材料骨架），gpt-image-2 只在可具象处出手——不把图像模型用在它不擅长的图解上。
- **显式触发**：控制 gpt-image-2 成本/时延，并保证图的质量（需精心提示词）。
- **设计为输入/输出 agent 的组合**：呼应课程方法论，便于 v2 扩成多 agent 工作流。

## 隐私边界

密钥只存插件本地 data.json、绝不入库。但内容会按下述方式出网，使用前需知悉：

- **笔记全文** → 嵌入代理（`embedBaseUrl`）：索引/重建索引时，vault 内每篇笔记的分块正文都会发去做嵌入。（这是从「本地嵌入」改为「云端嵌入」后的边界变化——原设计承诺笔记不外发，现已不成立。）
- **聊天上下文 + 检索片段** → 文本 LLM 代理（`llmBaseUrl`）：对话时把历史、当前问题、以及检索到的相关笔记片段发给 LLM。
- **配图提示词** → 图像代理（`imageBaseUrl`）：仅你的概念/提示词，不含笔记原文。

若对某些笔记敏感，当前没有按目录排除索引的开关（可作为后续增强）。

## 错误处理

- API 失败：优雅提示 + 重试，不丢对话。
- 配图：仅显式触发，避免烧钱；失败回退为纯文字。
- RAG 空命中：直说「无相关旧笔记」，照常讲。
- LLM 吐的 Mermaid 语法错：先校验 / 自动修复，失败则展示原文 + 警告。
- 写笔记：不覆盖，冲突加时间戳。
- 嵌入 API 失败：自动退避重试（429/5xx，最多 3 次）；仍失败则该文件本轮不入索引、以 Notice 提示，下次编辑或「重建索引」再试（不阻塞对话；检索维度不一致等错误在导师侧被吞掉、照常对话）。增量更新对编辑做防抖（~2.5s），避免编辑期间反复打嵌入接口。

## 测试策略

- **单测纯函数**：提示词拼装、Mermaid 校验、笔记模板渲染、检索排序、向量相似度。
- **集成**：在测试 vault 内手测全流程。
