# Cobrain 实践演练：跟着走一遍完整链路

> 这是一次**真实运行的实录**——用「世界何以至此」库现场跑出来的，检索命中、副脑回应、概念图、笔记，全部照搬，没有一句是我编的台词。
>
> **但先说清楚两件事，免得你照着做却对不上：**
> 1. **检索命中取决于你自己的 vault。** 下面撞出来的是这个库里的笔记；你的库不同，同一句话撞出的会是**你自己**写过的另一批东西。这正是它的价值，不是 bug。
> 2. **副脑的回应有随机性**（temperature 0.7）。同一句话问两次，措辞都会不一样。所以别把这篇当"标准答案脚本"去比对——要学的是**怎么读它给你的东西**，不是背它说了什么。
>
> 演练主题：**「总觉得要先实现财务自由，才能去做想做的事，却迟迟不敢动」**——一个谁都卡过的念头。

如果你读完《使用手册》还是觉得"道理我都懂，可上手不知道干嘛"，这篇就是给你的。Cobrain 的价值不在某个按钮，而在一条**链路**：抛困惑 → 撞出你写过的旧笔记 → 被回抛问题 → 自己想一步 → 沉淀成带双链的新笔记 → 下次被这篇撞出来。链路转起来，第二大脑才活。下面一格一格走。

---

## 场景 0 · 开演前：库里得有东西

Cobrain 的本事是"接住你写过的"。空库用它，约等于对着空气发问。

「世界何以至此」库里有约 900 篇笔记，密集分布在创作、哲学、思想、经济、社会、读书笔记这些方向。索引完，全库被切成约 9700 个片段（chunk）等着被你的问题撞上。

**前置动作**（只做一次）：命令面板（Ctrl/Cmd+P）→ `Cobrain: 重建索引`，等它把整库嵌入完。之后这库就"认得"你写过什么了。

> 从 0.1.3 起，**新存进来的笔记会自动入索引**（防抖约 2.5 秒），所以第一次重建之后，日常基本不用再手动重建——这点场景 6、7 会亲眼看到。现在也可以用命令 `Cobrain: 查看索引状态` 主动看已索引篇数、chunk 数、最近更新时间和失败列表。

## 场景 1 · 抛出一个真实的困惑

点左侧栏的**脑图标**（或命令 `Cobrain: 打开创作副脑`），右侧开出面板。开场白写着：

> 聊你正在想的。我会翻出你写过的相关旧笔记，并回抛问题逼你自己想。下方：概念图 / 推敲 / 配图 / 存为笔记。

> 如果你之前关过这个槽位的面板，打开时会先问你「恢复草稿」还是「新建空对话」。这不是多余步骤：关闭面板现在表示暂停，不表示删除思路。真的想结束这一轮，用面板里的「清空」。

在底部输入框（占位提示「问副脑…（Enter 发送，Shift+Enter 换行）」）里，敲下一句**大白话的真困惑**——不用组织成"好问题"，就是心里那句话：

```
我最近总卡在一个念头里：总觉得要先实现财务自由，才能真正去做自己想做的事。
可越这么想越焦虑，迟迟不敢动。帮我想想这是怎么回事。
```

回车。面板冒出"思考中…"。

## 场景 2 · 它先撞出你写过的旧笔记

回应出来之前，面板先摊开一块 **「相关旧笔记」**。它在后台把你这句话做了语义检索，从约 9700 个片段里捞出最相关的 6 个，列成可点的条目。这次真实撞出来的是：

```
相关旧笔记
─────────────────────────────
晓晨元宇宙：钱根本不是挣来的
  …要的事更要先做。
─────────────────────────────
20 关于工作与赚钱的一切
  多数人真正的问题并不是不知道自己擅长什么，而是他们无法克服「新手恐慌」…
─────────────────────────────
11 为什么听了这么多道理，却依然做不好投资？
  …我会定下一个量化的指标，比如一个月要跑 100 公里…最后在脑子里恣意幻想…
─────────────────────────────
做题家&出题家 › 螺旋式上升，才是普遍现象
  …还没完全想清楚的时候，这种焦虑就已经影响到自己的情绪…
─────────────────────────────
06 殖民历史中萌芽的美国代议制
  …给我，我就要追求独立了。
```

点任意一条，对应笔记就在主编辑区打开。

> **诚实的旁白（重要）：** 看第 5 条——`美国代议制`。它跟你的问题基本无关，是撞上了"独立"两个字才进来的。**这是语义检索的常态，不是坏了。** 命中分数往往挨得很近（这次前六名都在 0.63～0.66 之间），所以偶尔会混进一两条只是字面沾边的。
>
> 这恰恰说明为什么「相关旧笔记」这块要**你自己扫一眼**：6 条里有 4 条戳中了（赚钱、新手恐慌、跑步计划半途而废、焦虑），那就够了。**别指望它每条都精准，指望它把你忘了的相关笔记勾一两条到眼前**——那一两条，常就是钥匙。
>
> （另一个真相：你库里其实有更"对题"的笔记，比如 `自由的悖论：在限制中寻找自由`、`只有自律的人才能拥有自由`，但它们这次排在第 12、13 名，差一点没挤进前 6。换个问法——比如直接问"自由和限制的关系"——它们就会浮上来。检索是跟着你的**措辞**走的。）

## 场景 3 · 第一轮回抛：它不给答案，给问题

紧接着，副脑的回应到了。注意它**有多克制**——没有长篇大论，而是从你刚撞出的旧笔记里，挑出一句最锋利的，反手扣回你的问题上：

> **副脑：**
>
> 你写的那些笔记里，其实已经藏着一个很关键的矛盾。你自己记过：
>
> > 没创造力很可能不是你无法从事创意工作的原因，而是你没从事创意工作的结果。
>
> 那我把这句话的结构借过来问你一句：**"没钱"很可能不是你无法做想做的事的原因，而是你还没开始做想做的事的结果**——你觉得这个说法成立吗？如果成立，它成立在什么地方？如果不成立，卡在哪里？

> 旁白：拆开看它做了什么——
> 1. **接地**：它没从零讲道理，而是引用**你自己写过的一句话**（"没创造力是结果不是原因"），把你今天的困惑接到你旧的洞见上。
> 2. **辩证逆转**：它把你那句话的**结构**借走，套到"没钱"上——把你眼里的"前提条件"（财务自由）翻成了"结果"：会不会正是这个困扰，才让那件事成立？
> 3. **先问后讲**：到此为止。它没替你下结论，把"这成立吗、卡在哪"留给你。

**这里是关键动作：别跳过这个问题，先自己答一句再往下。** 被问住的地方，就是你认知里的缝。

## 场景 4 · 你答一句，它把你逼到更深一层

你被问住了，老实回它：

```
被你问住了……我好像确实不是缺钱，是怕开始之后做不好、坚持不下来。那我该怎么办？
```

副脑接住你这步——注意它**没有**直接给"怎么办"，而是又从你库里捞出一篇，把你的状态照给你看，再追一个更尖的问题：

> **副脑：**
>
> 你笔记里刚好有一段，几乎就是你现在状态的翻版——
>
> > 我会定下一个量化的指标，比如一个月要跑 100 公里……最后，在脑子里恣意地幻想，如果我真的变成了一个可以随便跑马拉松的运动健将，那感觉该有多好。但是事情通常会怎么发展呢？那就是，自己早上起不来……偶尔出去跑一次，又觉得自己体力很差，跑起来很痛苦。
>
> 你看到了吗？你现在的"先财务自由再做事"，和"先买齐装备定完计划再跑步"，是同一套逻辑。问题不在于你缺什么条件，而在于这套逻辑本身。
>
> 那我问你一个差异性的问题：**"跑步"和"做想做的事"，在你现在的想象里，有什么本质不同？为什么跑步你会承认自己是新手恐慌，但做事你却归因成了财务不自由？**

> 旁白：这一轮它做了件更狠的事——把你库里一篇**讲跑步**的笔记，和你**讲钱**的困惑，**并置**在一起，让你自己看出那是同一个结构。然后用**差异性发问**收口："为什么同一套逻辑，你对跑步诚实，对做事却自欺？" 它逼你直视的，是你的**归因方式**，不是"自由"这个抽象词。

来回几轮，你会发现：**它从头到尾没替你下过结论。** 它只是不停地把你写过的东西、和你此刻的卡顿，对撞在一起。

## 场景 5 · 让它起个头：概念图

思路有点多，想看它们怎么搭在一起。点面板顶部的 **「概念图」**（它拿当前话题去生成）。副脑吐出一张 Mermaid 图：

```mermaid
graph TD
    A["先财务自由，再去做想做的事"]
    A --> B[核心矛盾]
    B --> C["倒果为因"的认知陷阱]
    C --> D["新手恐慌"与认知防火墙]
    D --> E[物质匮乏的恐惧]
    D --> F[心理胜任的恐慌]
    F --> G["完美准备"的拖延]
    G --> H["做题家"的线性思维]
    H --> I[焦虑的恶性循环]
    I --> J[忽略"螺旋式上升"]
    J --> K[切换而非放弃]
    K --> L[先做重要的事]
    L --> M[在实践中积累胜任感]
    M --> N[打破"先财务自由"的执念]
    C -.->|源自| O["钱根本不是挣来的"]
    D -.->|详见| P["关于工作与赚钱的一切"]
    G -.->|类似| Q["为什么听了这么多道理却做不好投资"]
    H -.->|参考| R["做题家&出题家"]
```

> 旁白：这张图把整场对话的骨架立了起来，还用虚线标了每个节点来自你哪篇笔记。**但别把它当成品**——使用手册那句话："概念图：让它起头，你来连。" 价值在你盯着它、发现"这条线串得对不对""这里是不是缺一环"、然后自己动手改的过程。它给草稿，你做建构。
>
> （提示：模型有时会在箭头标签里塞 `[[双链]]`，那样 Mermaid 会渲染失败——遇到了就把标签里的双链改成普通文字。这次它没犯，图能直接渲染。）

## 场景 6 · 存为笔记：闭环

这一趟想清楚了，别让它蒸发。点 **「存为笔记」**。副脑不会把聊天记录原样存下来，而是**综述成一篇结构化笔记**，自动双链回刚才撞出的旧笔记。几秒后弹出"已保存：cobrain-note/……"。存下来的真实内容：

> 保存时会冻结这一刻的对话、来源、概念图和配图。你在弹窗期间继续聊天，后面的内容不会悄悄混进这篇笔记。同一份内容再点一次，会提示已经保存过；如果你又生成了新的概念图、配图或继续聊出了新来源，它会明确当作新版本保存。

> **标题：从"先财务自由"到"先行动"的认知拆解**
>
> ### 一、核心矛盾：因果倒置的思维陷阱
> - **常见念头**：必须先实现财务自由，才能做想做的事。
> - **关键反转**（借用笔记中的句式）：原句"没创造力是你没从事创意工作的结果，而非原因"，套用为"没钱是你还没开始做想做的事的结果，而非原因"。
> - **结论**：问题不在缺钱，而在用"财务不自由"掩盖对行动本身的恐惧。
>
> ### 二、恐惧的实质：对过程而非条件的回避
> - **真实障碍**：怕开始后做不好、怕坚持不下来。
> - **类比揭示**（来自跑步计划的笔记）：定量化指标 → 幻想成功状态 → 现实受挫 → 放弃。当前模式：定财务目标 → 幻想自由生活 → 焦虑 → 不敢动。同一套"完美准备"思维。
>
> ### 三、差异性问题：为何归因不同？
> - 跑步：承认是"新手恐慌"，问题在自身适应过程。
> - 做事：归因为"财务不自由"，把内在恐惧外化为客观条件。
> - **破解点**：把"做想做的事"重新定义为可渐进习得的过程，而非一次性达标的状态。
>
> ### 四、行动启示
> - **打破因果倒置**：开始行动本身会创造条件和能力，而非等条件齐备再动。
> - **接受新手阶段**：允许自己做不好、坚持不稳，视为必经阶段而非失败证据。
> - **最小启动**：不追求宏大计划，从可立即执行的微小行动切入，积累正反馈。

笔记头部自动带上 `tags` 和双链来源；如果你刚才画了概念图，它也会一并嵌进来。

> 旁白：注意这篇笔记**不是那几篇旧笔记的复读**——它是你这趟思考的**新产物**，把"没钱是结果不是原因""跑步计划的循环""新手恐慌"拧成了一条可行动的结论。这就是"比照—落差—改进"循环里沉淀下来的东西。

## 场景 7 · 第二天：闭环真正合上

存完这篇，**它在约 2.5 秒后自动进了索引**（0.1.3 起新建笔记自动入索引，不用手动重建）。隔天你想别的，比如：

```
我想做个副业但迟迟没开始，是不是又在等"准备好"？
```

面板的 **「相关旧笔记」** 里，**昨天那篇刚存的笔记被撞了出来**——"从'先财务自由'到'先行动'的认知拆解"。你昨天的思考，今天回来接住你。

> 旁白：这就是链路**合上**的一刻。抛困惑 → 撞旧笔记 → 被回抛 → 自己想 → 存成新笔记 → **新笔记又成为下一次的旧笔记**。日积月累，vault 不是越堆越乱的仓库，而是一个会主动接住你、跟你对话的第二大脑。
>
> 想验证它真入索引了？命令 `Cobrain: 测试检索`，输入"先行动 新手恐慌"，那篇新笔记应当出现在命中里（带分数、路径、片段）。
>
> 如果你怀疑某篇没入索引，先用 `Cobrain: 查看索引状态` 看最近失败列表和更新时间，再决定要不要重建索引。别一上来就全量重建，vault 大时这很浪费。

## 一个进阶动作 · 边读边问：「引用进 Cobrain」

读旧笔记时撞到一句想深挖的话，不必手抄。**选中那段文字 → 右键 → 「引用进 Cobrain」**（编辑模式下可用）。选区连同它的来源链接、所在小节，会预填进面板输入框，你补一句问题发出去。副脑这一问会额外带上**你正在读的那段上下文**，回应更贴你眼前的语境。

## 复盘 · 这一趟你真正得到了什么

不是"该不该先攒钱"的标准答案。是：

- 你把一个模糊的焦虑（"要先有钱才能做事"），**亲手**翻成了一个能看清的真相（"我怕的不是没钱，是开始之后做不好"）。
- 你写过的笔记被重新激活、并联，拧成了一篇新的。
- 这篇新的，明天会回来找你。

副脑全程只做了一件事：**逼你想，不替你想**。它是顺手的厨房，不是摆功能的客厅。

---

## 为什么你照着做，结果会不一样（必读）

这不是文档骗你，是这工具的**机制**决定的：

- **撞出的笔记不同**：检索跟着你的 vault 和你的**措辞**走。你的库里没有"跑步计划"那篇，就撞不出它；你把问题问得更抽象/更具体，命中也会整批变化。
- **副脑的话不同**：同一句话两次问，回应措辞都不同（有随机性）。它的"招法"稳定（差异性发问、辩证逆转、先问后讲），但具体说什么不固定。
- **所以正确的用法是**：把上面这场对话当**范例**，学它的**节奏**——你抛真困惑、它撞旧笔记+回抛、你别跳过它的问题、来回几轮、存成笔记。至于它具体撞出你哪篇、问你哪句，交给你自己的 vault。

## 反模式 · 怎么把这插件用废

- **当搜索引擎使**：直接要标准答案——那不如查百科。它的价值在回抛，不在应答。
- **跳过它的问题**：它回抛你不答、只追"那结论呢"——循环就断了，等于把助产士当讲解员。
- **空库 / 薄库用**：没有前知识可接，它撞不出你写过的东西，只能泛泛而谈。**先写，再用它撞。**
- **嫌命中不精准就不用**：top 6 里有一两条沾边的很正常，扫一眼挑有用的就行——别因为它没做到"每条都准"而否定"勾出关键一两条"的价值。
- **把概念图当成品**：一键生成就完事、不动手改——你跳过了唯一产生理解的环节。
- **囤积而不消化**：狂存笔记当收藏。"忘了就忘了，说明它不重要"——别把知识管理做成囤积癖。

| 给概念配图 | 顶部「配图」→ 选概念 → 编辑提示词 → 出图（约 1 分钟） |
| 沉淀成笔记 | 顶部「存为笔记」→ 自动综述 + 双链，~2.5s 后自动入索引 |
| 边读边问 | 选中文字 → 右键「引用进 Cobrain」 |
| 验证检索 | 命令 `Cobrain: 测试检索` → 输一句话看命中 |
| 查看索引健康度 | 命令 `Cobrain: 查看索引状态` → 看篇数、chunk、更新时间、失败列表 |
| 换了嵌入模型后 | 命令 `Cobrain: 重建索引`（仅桌面端） |
